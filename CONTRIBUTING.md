# Contributing

## Principles

Keep the system hackable.

Keep the playback core stable.

Prefer simple solutions over clever ones.

Design for experimentation.

## Architecture rules

Audio must not depend on UI.

Input must not contain product logic.

Services must communicate via defined interfaces.

## Code rules

Small services.
Clear interfaces.
Readable code.

## Documentation rule

Architecture changes require an ADR.

Major changes must update architecture docs.
